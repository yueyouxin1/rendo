# Rendo 新定位总纲

## 一句话定义

**Rendo 不再是一个构建平台，也不再以 DSL 为核心交付物；Rendo 的核心是面向强 Agent 的高质量 AI 产品 starter 资产体系。**

## 新定位的五个核心判断

### 1. 主语从“平台”变成“starter”

真正的产品核心不是平台本身，而是：

- 可运行
- 可维护
- 可被强 Agent 持续二开

的 starter。

### 2. 默认最优解比“绝对开放”更重要

Rendo 不需要把一切开放到最小原子层。  
更重要的是提供：

- 默认可运行
- 默认可扩展
- 默认少踩坑

的起点。

### 3. 运行时层应优先采用成熟主流方案

Rendo 不应继续把这些底层能力作为主要自研目标：

- Agent runtime
- Workflow engine
- DB / Auth / Billing runtime

关键点不在于“它们是否通用”，而在于：

- 这些层已经存在更成熟、更主流、被更多用户与 Agent 认可的做法
- 继续从零自研，长期很难形成对成熟生态的持续优势

因此，Rendo 的正确做法是：

- 优先采用成熟主流方案
- 在其之上组织默认最佳实践
- 只保留默认最优体验所必需的最小自研控制层

### 4. 多端不靠一套前端打穿

统一一致体验应通过：

- 共享核心
- 多前端壳
- 统一后端

来实现。

### 5. 平台功能从 starter 反向长出

模板分发、升级、授权、治理等平台功能，不应先做，而应从 starter 的真实使用中反向抽象。

## 当前最重要的事

不是先做平台，而是：

- 定义 `Core Starter`
- 做出第一个高质量 `Starter Template`
- 建立自然的 `Capability Template` 机制
