# Rendo TODO 目录说明

本目录按阶段拆分 TODO，不是单个总表。

目录结构：

- `core-starter/`
  - 负责从契约、底座到第一个 starter 与能力模板机制的实现
- `rendo-platform/`
  - 负责双模式、共享核心、多端准备和平台闭环

建议执行顺序：

1. `core-starter/00-Phase0-契约与规范TODO.md`
2. `core-starter/01-Phase1-CoreStarter实现TODO.md`
3. `core-starter/02-Phase2-RendoCLI实现TODO.md`
4. `core-starter/03-Phase3-第一个DomainStarter实现TODO.md`
5. `core-starter/04-Phase4-Pack机制实现TODO.md`
6. `rendo-platform/10-Phase5-最小真实Rendo能力接入TODO.md`
7. `rendo-platform/11-Phase6-双模式与多端准备TODO.md`
8. `rendo-platform/12-Phase7-Rendo平台闭环TODO.md`

一句话：

**先把 `Core Starter`、`CLI`、`Starter Template` 和 `Capability Template` 做成，再让平台长出来。**
