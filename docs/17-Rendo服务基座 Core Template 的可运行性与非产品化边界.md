# Rendo 服务基座 Core Template 的可运行性与非产品化边界

## 结论

`core` 模板应至少可做最小健康校验，但不应演变成完整产品模板或完整服务基座交付物。

## 为什么要可运行或可校验

因为如果 core 层完全不可验证，就无法确认：

- manifest 是否成立
- 项目清单是否成立
- CLI 初始化结果是否成立
- Agent 能否稳定理解当前模板
- `.agent` / `api` / `mcp` / `skills` 的基本入口是否存在

## 为什么不能产品化

因为 `core` 的职责是定义控制面，不是提前做完 `base` 或 `derived`。

产品形态、业务模块、具体厂商绑定，都应该上移到更高层。
