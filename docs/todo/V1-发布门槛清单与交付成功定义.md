# V1 发布门槛清单与交付成功定义

## 结论

在当前仓库阶段，任何 `v1` 讨论都必须先通过一个更前置的门槛：

- **Rendo 必须先把“面向 Agent 的服务基座”首日架构标准冻结并落实到 `core -> base -> derived`**

如果这一步没做完，就不应继续把远程 runtime、狗粮验证或更大平台面视为第一顺位。

---

## 一、当前阶段的第一发布门槛

### A. 首日架构与目录标准

- [x] 已有正式的首日架构与目录标准文档
- [x] `README.md` 与 `AGENTS.md` 已与该标准对齐
- [ ] 五类 `core` 模板都对齐同一套语言无关目录语义
- [ ] 所有模板都对齐 `AGENTS.md / CLAUDE.md / .agents / interfaces / src / tests / scripts / install`
- [ ] `starter-core-template` 已提供宿主挂载位与 `ops/` 语义
- [ ] 非 starter `core` 模板已明确宿主扩展模型，而不是只给散乱文件
- [ ] `base` 模板已把该标准落实为主流、生产可演进的最佳实践
- [ ] `derived` 模板继承规则已明确不可破坏标准目录与 Agent 入口

### B. Agent 可调用标准

- [ ] 业务能力可以稳定映射到 `.agents/capabilities.yaml`
- [ ] Starter 基座明确保留 `interfaces/openapi / interfaces/mcp / interfaces/skills` 三类接口描述面
- [ ] 非 starter 模板安装时会显式说明它更新了哪些 Agent 入口与接口描述面
- [ ] CLI / 检查脚本能够暴露这些能力入口，而不是让 Agent 猜测

### C. 测试、验证与运行等级

- [ ] `core` 已提供 TDD 导向的测试骨架
- [ ] 所有模板都可做结构、契约与集成验证
- [ ] `standalone-runnable` 模板已提供启动、smoke、健康检查或等价心跳机制
- [ ] 模板安装和升级会验证对 `AGENTS.md / .agents / interfaces/*` 的影响
- [ ] 结构漂移可以被自动检查发现

### D. CLI、模板产物与生命周期

- [x] `rendo init <kind>` 正式可用
- [x] `rendo create` 正式可用
- [x] `rendo search / inspect / pull` 已支持远程 registry
- [x] `rendo add / upgrade` 至少对首批官方资产正式可用
- [ ] CLI 输出已完整体现首日架构标准所需的关键信息
- [ ] `shared/authoring/templates` 与 `shared/templates` 的 authoring/产物边界已被脚本、文档和流程共同约束
- [ ] 至少一个非 starter 官方模板已证明完整 `add / pull / install` 闭环
- [ ] CLI 的正式发布形态已明确，不再把“自包含二进制”误当成当前阶段前置条件

## 二、当前阶段交付成功的定义

以下条件同时满足，才算当前阶段交付成功：

1. 一个新用户或强 Agent 可以从 `core` 模板直接读出统一的服务基座目录语义，而不是依赖隐式约定。
2. 每个 `core` 模板从第一天就明确保留 `AGENTS.md / CLAUDE.md / .agents / interfaces / src / tests / scripts / install`。
3. `application-base-starter` 已把这套结构落实为第一套生产可演进的官方服务基座 base，而不是 demo scaffold。
4. `feature / capability / provider / surface` 模板都能明确说明自己如何挂到宿主 starter 的标准目录、Agent 入口与接口描述面。
5. `shared/authoring/templates` 与 `shared/templates` 的边界对作者、CLI 和 registry 都清晰无歧义。
6. 强 Agent 不需要猜“能力定义在哪、接口描述在哪、该修改哪些目录、该验证哪里”。
7. CLI、结构校验脚本或同步检查能够验证这套标准没有漂移。

---

## 三、不满足就不能往下推进的情况

出现以下任一情况，都不应把 runtime 或平台线提到第一顺位：

- 五类 `core` 模板目录语义仍然不一致
- `starter-core-template` 还没有标准宿主结构
- 非 starter 模板还没有宿主扩展边界
- `base` 仍然只是框架样例，而不是生产可演进服务基座
- 模板安装对 `AGENTS.md / .agents / interfaces/*` 的影响仍然是隐式的

---

## 四、完成上述门槛之后再推进的东西

以下内容在当前阶段不是第一顺位，但可以作为后续阶段继续推进：

- headless runtime
- 模板远程发布与持久化存储
- 自包含 CLI 分发
- Rendo 自身狗粮验证
- 更完整的生产运维闭环
- 平台前端、控制台和商业化界面
