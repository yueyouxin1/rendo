# 多端体验与统一后端原则

## 核心结论

统一一致的多端体验，不应通过“一套前端实现打穿所有端”来获得，而应通过：

- 共享核心
- 多前端壳
- 统一后端

来获得。

## 默认推荐结构

### Web

- `Next.js`

### 小程序

- `Taro`

### App

- `Expo / React Native`

### 桌面端

- `Tauri`

### 统一后端

- `Next.js` 作为 Web 主应用和统一业务后端 / BFF / AI 后端

### 重任务

- `Trigger.dev` / worker

## 共享什么

应共享：

- contracts
- domain
- sdk
- config
- validation
- agent / workflow clients

不应强行共享：

- UI 页面
- 布局层
- 路由层
- 端特定交互

## 当前阶段原则

第一版仍然：

- `Web-first`

但必须从第一天就为：

- 小程序
- App
- 桌面端

预留共享核心结构。

## 最终原则

**统一一致体验的关键是“统一语义和统一后端”，不是“统一 UI 实现”。**
