# 产品门面与商业化体验 TODO

## 目标

默认门面必须直接对标专业云服务商 SaaS 模板体验。

## 视觉与风格

- [ ] 以 Vercel 官网作为默认 UI 门面的直接参考
- [ ] 前端统一采用 `vercel/shadcn` 极简风格
- [ ] 默认使用 `shadcn` 浅色主题
- [ ] 支持多种色彩主题切换
- [ ] 统一产品字体、间距、层级、阴影、圆角与动效规范

## 默认门面页

- [ ] 交付专业的 marketing / landing 页面
- [ ] 交付价格页与订阅方案页
- [ ] 交付登录 / 注册 / onboarding 流程
- [ ] 交付 dashboard 首页
- [ ] 交付项目列表 / 项目详情门面
- [ ] 交付账单 / 订阅 / 使用量页面
- [ ] 交付帮助、错误、空状态和加载状态

## 商业化体验

- [ ] 默认具备 plan / entitlement / usage / billing 概念
- [ ] 默认具备订阅变更入口
- [ ] 默认具备结账、升级、降级、试用和取消的 UI 流程
- [ ] 所有未配置商业化能力都必须弹出清晰、优雅、可操作的错误提示
- [ ] 禁止静默失败、禁止“点了没反应”

## 完成标准

- [ ] 默认产品门面看起来像专业、整洁的云服务商 SaaS 模板
- [ ] 商业化功能的 happy path 与 failure path 都有可见、可理解反馈
