# Rendo 服务基座 Core Template 的可运行性与非产品化边界

## 结论

`core` 模板必须全部可验证，但不要求全部都是独立运行服务。

更准确的边界是：

- 所有 `core` 模板都必须可做最小结构与契约校验
- 只有被定义为 `standalone-runnable` 的 `core` 模板，才要求最小启动与健康检查
- `core` 不应演变成完整产品模板或完整服务基座交付物

## 为什么必须可验证

如果 core 层完全不可验证，就无法确认：

- manifest 是否成立
- 项目清单是否成立
- CLI 初始化结果是否成立
- Agent 能否稳定理解当前模板
- `AGENTS.md`、`.agents/`、`interfaces/*` 等关键入口是否存在

## 为什么不要求全部独立运行

因为很多非 starter core 模板本质上是：

- `host-attached`

它们的价值在于：

- 定义装配边界
- 定义宿主影响面
- 定义可验证的最小契约

而不是单独作为一个服务进程长期存在。

## 为什么不能产品化

因为 `core` 的职责是定义控制面，不是提前做完 `base` 或 `derived`。

产品形态、业务模块、具体厂商绑定，都应该上移到更高层。
