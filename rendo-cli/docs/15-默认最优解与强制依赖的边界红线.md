# 默认最优解与强制依赖的边界红线

## 核心结论

Rendo 可以提供默认最优解，但不能把默认最优解做成事实上的强制依赖。

这条红线适用于所有模板类型，不只适用于 starter。

## 红线 1：Core 模板不能有硬平台依赖

如果某个 `core` 模板没有平台 key 就无法初始化、校验或被 Agent 理解，
那它就不是 core 层，而是平台客户端。

## 红线 2：Base 模板不能锁死退出路径

官方 `base` 可以更完整，但必须保留：

- 替换 provider 的路径
- 替换 capability 的路径
- 退出托管能力的路径

## 红线 3：Capability / Provider 模板不能侵占宿主主权

通过 `add` 装入 starter 的模板，不能偷偷接管整个宿主工程。
