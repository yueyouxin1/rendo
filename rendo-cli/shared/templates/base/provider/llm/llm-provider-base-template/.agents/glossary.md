# Glossary

- `core`: freezes the Rendo engineering language for a template kind.
- `base`: official reference implementation of the matching core.
- `derived`: concrete product, scenario, or community template built on the frozen core/base contract.
- `control plane`: metadata, docs, agent entrypoints, and interface surfaces that define how the workspace is understood.
- `implementation root`: `src/`, the only default location for implementation code.
- `interface surface`: `interfaces/openapi`, `interfaces/mcp`, or `interfaces/skills`.
- `host-attached`: template or asset that depends on a host workspace to run.
- `standalone-runnable`: template that must boot, smoke-test, and expose health or equivalent heartbeat semantics.
